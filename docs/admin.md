# Админ-панель — возможности менеджера и учителя

> **Назначение.** Что редактируется из админки (django-unfold), как правки попадают на витрину и какие операции закрыты намеренно.
> **Статус.** Актуальный. Авторитет по админке.
> **Связанные документы.** `project-context.md` (бизнес-правила), `schema.md` (структура данных), `api-core-contracts.md` (публичные эндпоинты).

Все правки витрины видны на сайте **мгновенно**: инвалидация payload-кэша висит на сигналах `post_save`/`post_delete` в `apps/public_api/signals.py`. Работает из админки, shell и сервисов; исключение — `QuerySet.update()`, он сигналы обходит, такие правки дожидаются TTL (≤ 5 мин).

## Тарифы (Биллинг → Тарифные планы)

- Цена, базовая цена занятия, число занятий — редактируются прямо в списке.
- `is_unlimited` — плашка «Безлимит» (без цены за занятие, уходит в конец витрины), `is_active` — скрыть/показать тариф.
- Цена за занятие на карточке считается автоматически: `round(price / slots_count)`.
- Смена цены безопасна для уже купленных абонементов: у них зафиксирован `purchase_price`.
- Отдаётся фронту через `GET /api/v1/public/plans/`.

## Кружки (Каталог → Activity)

Название, слуг (автозаполнение), категория, цена, обложка (URL из S3), краткое и полное описание, особенности и теги (JSON-списки), вкл/выкл. Влияет на топ-3, полный каталог и детальные карточки.

## Расписание (Расписание → Группы)

Группа: кружок, преподаватель, кабинет, слот, возраст от/до, вместимость, вкл/выкл (возраст и вместимость — прямо в списке). Плюс справочники кабинетов и временных слотов.

Переносы и отмены занятий (маски) в админке **не редактируются намеренно** — только через сервис `create_schedule_mask`: он валидирует коллизии кабинетов и преподавателей под advisory-локами, прямое создание в админке обошло бы эти проверки.

## Преподаватели (Пользователи → Профили преподавателей)

Привязка к пользователю, отчество, фото (URL), должность на витрине, цитата, био. Список на сайте собирается из активных групп: преподаватель без активных групп с витрины исчезает сам.

## Галерея (Контент → Фото галереи)

URL фото, порядок и публикация — прямо в списке.

## События (События)

- Событие: название, описание, обложка, дата, длительность, цена, вместимость, публикация; `seats_taken` — readonly.
- Регистрации: контакты, источник и комментарий редактируются; создание и удаление запрещены, смена статуса — только экшенами «Подтвердить оплату» / «Отменить и освободить места». Это защита инварианта `seats_taken`: ручная правка статуса рассинхронизировала бы счётчик мест.

## Заявки с сайта (Публичные формы)

Обратный звонок и обратная связь: смена статуса обработки прямо в списке, история изменений — от `django-simple-history`.

## Клиенты (Пользователи) — карточка семьи

Родитель (поиск по email/телефону) с инлайнами: дети и абонементы (read-only, создание закрыто — абонементы появляются только через чекаут). Комментарии учителей по ребёнку — «Журнал → Комментарии», поиск по ФИО ребёнка.

## Журнал (роль «Учителя»)

- Учитель — staff-пользователь в группе «Учителя» (группа создаётся миграцией `journal/0002`) с привязанным TeacherProfile. Видит **только свои** группы.
- «Занятия»: фильтр Сегодня/Прошедшие/Будущие, тема занятия прямо в списке, инлайн комментариев по детям (тональность +/0/−, автор проставляется автоматически).
- «Посещаемость»: отметки создаются автоматически утренним таском (все записанные — «пришёл»), учитель переключает статус прямо в списке; создание и удаление закрыты.
- Смена статуса посещения автоматически списывает или возвращает фишку абонемента — одной транзакцией с отметкой.
- Админ видит все группы, все комментарии и посещаемость.

## Дашборд (главная админки)

Выручка месяца, активные абонементы, должники по возвратам, выручка по месяцам (6 мес.), самые пустые группы, кружки по числу записанных. Источник: `apps/core/dashboard.py` + `templates/admin/index.html`. Палитра Unfold — фирменный зелёный.

## Массовые операции

«Заморозка абонементов» (каникулы, карантин): выбор абонементов в списке → экшен с периодом и причиной, срок действия сдвигается на длину заморозки одним пакетом в одной транзакции.

## Что не редактируется из админки (осталось на фронте)

Hero-тексты («Найдите занятие...»), «От 450 ₽ за занятие», «пробное — 1 200 ₽», «максимальная выгода 50 %», «абонемент оформляется на месяц». Если нужно управлять и этим — отдельный этап: singleton-модель SiteSettings в `apps/content` + `GET /api/v1/public/settings/` (решение за владельцем проекта).
