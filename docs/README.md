# Документация проекта «Улица Радости»

Это папка с проектной документацией детского центра «Улица Радости» (сайт + личный
кабинет). Здесь нет кода — только описание *что* мы строим, *по каким правилам* и
*как устроены* данные, авторизация, покупки и API.

**Если ты тут впервые — начни с `project-context.md`.** Он объясняет бизнес простыми
словами. Остальное — технические детали поверх него. Реализация живёт в корне этого
же репозитория; при расхождении дока и кода прав код, а док чинится.

---

## С чего начать (по ролям)

| Ты кто | Читай в этом порядке |
| ------ | -------------------- |
| **Директор / заказчик** | `project-context.md` — что за продукт и правила бизнеса; `admin.md` — что можно менять самим |
| **Новый разработчик** | `project-context.md` → `schema.md` → `api-core-contracts.md` → нужный флоу |
| **Фронтендер** | `project-context.md` → `api-core-contracts.md` → `auth-flow.md` → `checkout-flow.md` |
| **Бэкендер** | `schema.md` → `schema-audit.md` → `schema-refactoring.md` → `api-core-contracts.md` |

## Кто за что отвечает (единственный авторитет на тему)

Чтобы не было противоречий, **у каждой темы один авторитетный файл**. Меняешь тему —
правишь её файл, в остальных оставляешь только ссылку.

| Тема | Авторитет |
| ---- | --------- |
| Бизнес-правила, продукт | `project-context.md` |
| Структура БД | `schema.md` (+ изменения в `schema-refactoring.md`) |
| HTTP-контракты, ошибки, вебхуки | `api-core-contracts.md` |
| Вход / регистрация / доступ | `auth-flow.md` |
| UX оформления покупок | `checkout-flow.md` |
| Публичные формы лидогенерации | `public-forms-design.md` |
| Админка | `admin.md` |

## Соглашения по стилю (для тех, кто пишет доки)

1. **Имена файлов** — `kebab-case`, латиница, `.md` (например `checkout-flow.md`).
   Без пробелов, кириллицы и заглавных.
2. **Каждый файл открывается шапкой** из трёх строк-цитаты:
   ```
   # Заголовок — назначение

   > **Назначение.** Одно предложение: о чём документ.
   > **Статус.** Актуальный / Архив + кто авторитет.
   > **Связанные документы.** Ссылки на соседние файлы.
   ```
3. **Манера подачи** — сухо и по делу. Перечисления — таблицами. Схемы состояний и
   потоков — блоками `mermaid` или ASCII. Решения — с обоснованием «зачем», а не только «что».
4. **Ссылки между файлами** — по имени файла в обратных кавычках (`auth-flow.md`),
   при необходимости с номером раздела (`project-context.md` §6).
5. **Деньги** — везде в копейках (целые числа). **Расписание** — день недели (0=Пн … 6=Вс) + время.
6. **Один источник правды на тему** (см. таблицу выше). Дубли запрещены — вместо копии ставь ссылку.

## Глоссарий (коротко)

Полный — в `project-context.md` §12.

| Термин | Значение |
| ------ | -------- |
| **Слот** | Регулярное занятие в сетке: день недели + время |
| **Абонемент** | Месячная покупка набора слотов для одного ребёнка |
| **Фишка** | Единица оплаченного посещения (1 слот = 4 фишки) |
| **Маска** | Разовый перенос/отмена занятия на конкретную дату |
| **Пробное** | Разовый визит; не более 1 на ребёнка по кружку |
| **Ивент** | Разовое мероприятие с фиксированным числом мест |
